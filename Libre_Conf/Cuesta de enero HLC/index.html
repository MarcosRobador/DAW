¿Qué es Git?

Git es un sistema de control de versiones distribuido, diseñado para manejar desde proyectos pequeños hasta muy grandes con velocidad y eficiencia. Fue creado por Linus Torvalds en 2005 para el desarrollo del kernel de Linux.

¿Para qué se utiliza?

Git

Ventajas de Git:

Distribuido: Cada desarrollador tiene una copia local del historial de desarrollo, lo que permite trabajar de manera descentralizada. Rendimiento: Git es rápido y eficiente en términos de rendimiento. Seguridad: Utiliza criptografía SHA-1 para asegurar la integridad del código fuente. Flexibilidad: Soporta varios flujos de trabajo no lineales y es eficiente en proyectos grandes y pequeños.

Comandos Básicos de Git:

git init: Inicializa un nuevo repositorio de Git. git clone [url]: Clona un repositorio existente. git add [archivo]: Agrega archivos al área de preparación para el próximo commit. git commit -m "[mensaje descriptivo]": Realiza un commit de los archivos que se han añadido. git push: Sube los cambios al repositorio remoto. git pull: Actualiza el repositorio local con los últimos cambios del remoto. git branch: Lista, crea o elimina ramas. git checkout [nombre_de_rama]: Cambia a otra rama.

¿Qué es Docker?

Docker es una plataforma de contenedores que permite empaquetar aplicaciones y sus dependencias en un contenedor virtualizado que puede ejecutarse en cualquier sistema Linux. Esto facilita la implementación y escalabilidad de aplicaciones, además de asegurar la consistencia en múltiples entornos de desarrollo, pruebas y producción.

¿Para qué se utiliza?

Docker se utiliza para crear, desplegar y ejecutar aplicaciones usando contenedores. Los contenedores permiten a los desarrolladores empaquetar una aplicación con todas las partes que necesita, como bibliotecas y otros dependencias, y enviarlo como un solo paquete.

Ventajas de Docker:

Consistencia y Aislamiento: Cada contenedor opera de manera aislada, lo que asegura la consistencia en todos los entornos. Portabilidad: Los contenedores pueden ejecutarse en cualquier sistema que tenga Docker instalado, lo que facilita la migración de aplicaciones. Eficiencia: Los contenedores comparten el mismo núcleo del sistema operativo, pero se ejecutan como procesos aislados, lo que los hace más ligeros que las máquinas virtuales tradicionales. Despliegue Rápido: Los contenedores pueden crearse, iniciarse y detenerse en segundos.

Comandos Básicos de Docker:

docker run [imagen]: Crea y ejecuta un contenedor a partir de una imagen. docker build -t [nombre]:[tag] .: Construye una imagen a partir de un Dockerfile. docker images: Lista las imágenes disponibles. docker ps: Muestra los contenedores en ejecución. docker stop [contenedor]: Detiene un contenedor en ejecución. docker rm [contenedor]: Elimina un contenedor. docker rmi [imagen]: Elimina una imagen.
